# Database

## 1. Overview

- **Database:** PostgreSQL
- **Extension:** `pgcrypto` — provides `gen_random_uuid()` used as default primary key for all tables
- **Total tables:** 11

All primary keys are UUID type, auto-generated at the database level. All timestamps use `timestamptz` (timezone-aware).

---

## 2. Tables Overview

| Table | Group | Description |
|-------|-------|-------------|
| `users` | Auth | User accounts (email/password and OAuth) |
| `conversations` | Chat | Chat sessions belonging to a user |
| `messages` | Chat | Individual messages within a conversation |
| `conversation_summaries` | Chat | LLM-generated summary of a conversation's older messages |
| `user_facts` | Chat | Personal key-value facts the agent remembers per user |
| `presentations` | Presentation | A presentation (stores current version) |
| `presentation_pages` | Presentation | Pages of the current version of a presentation |
| `presentation_versions` | Presentation | Archived previous versions of a presentation |
| `presentation_version_pages` | Presentation | Pages belonging to an archived version |
| `token_blacklist` | Auth | Revoked refresh tokens (used on sign out) |
| `password_reset_tokens` | Auth | One-time tokens for password reset via email |

## 3. Entity Relationship Diagram

```
  users
    │
    ├──(1:N)── conversations ──(1:N)── messages
    │               │
    │               ├──(1:1)── conversation_summaries
    │               │
    │               └──(1:N)── presentations
    │                               │
    │                               ├──(1:N)── presentation_pages
    │                               │          (current version pages)
    │                               │
    │                               └──(1:N)── presentation_versions
    │                                               │
    │                                               └──(1:N)── presentation_version_pages
    │                                                          (archived version pages)
    │
    ├──(1:N)── user_facts
```

> `conversations.active_presentation_id` → `presentations.id` (`ON DELETE SET NULL`)
> All other foreign keys use `ON DELETE CASCADE`

## 4. Table Details

### 4.1. users

Stores user accounts. Supports both email/password and OAuth providers.

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid PK | Auto-generated UUID |
| `email` | varchar(255) | Unique email address |
| `hashed_password` | varchar(255) | Bcrypt hash; NULL for OAuth-only accounts |
| `name` | varchar(255) | Display name |
| `avatar_url` | varchar(500) | Profile picture URL |
| `provider_user_id` | varchar(255) | User ID from OAuth provider (e.g. Google) |
| `email_verified` | boolean | Default `false`; set to `true` after OAuth login |
| `providers` | text[] | Array of auth methods, e.g. `['email', 'google']`; default `['email']` |
| `created_at` | timestamptz | Default `now()` |
| `updated_at` | timestamptz | Default `now()` |

---

### 4.2. conversations

A conversation session. Each user can have many conversations.

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid PK | Auto-generated UUID |
| `user_id` | uuid FK | Owner of the conversation |
| `title` | text | Auto-generated from first message; nullable |
| `active_presentation_id` | uuid FK | Points to the currently active presentation for editing; nullable |
| `created_at` | timestamptz | Default `now()` |
| `updated_at` | timestamptz | Default `now()` |

---

### 4.3. messages

Individual chat messages within a conversation.

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid PK | Auto-generated UUID |
| `conversation_id` | uuid FK | Conversation this message belongs to |
| `role` | text | `'user'`, `'assistant'`, or `'system'` |
| `content` | text | Message text content |
| `intent` | text | `'GENERAL'` or `'PPTX'`; NULL for user messages |
| `is_in_working_memory` | boolean | Default `true`; set to `false` when summarized |
| `summarized_at` | timestamptz | Timestamp when the message was summarized; NULL if still in working memory |
| `metadata` | jsonb | Extra data for PPTX messages: `slide_id`, `pages`, `topic`, `total_pages` |
| `created_at` | timestamptz | Default `now()` |

---

### 4.4. conversation_summaries

Stores the rolling LLM-generated summary for a conversation. At most one summary per conversation (enforced by UNIQUE on `conversation_id`).

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid PK | Auto-generated UUID |
| `conversation_id` | uuid FK UNIQUE | One summary per conversation |
| `summary_content` | text | The summary text generated by the LLM |
| `created_at` | timestamptz | Default `now()` |

---

### 4.5. user_facts

Personal information the agent remembers per user, stored as key-value pairs. A user cannot have two facts with the same key.

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid PK | Auto-generated UUID |
| `user_id` | uuid FK | Owner of the fact |
| `key` | text | Fact name, e.g. `"name"`, `"city"` |
| `value` | text | Fact value, e.g. `"Bao"`, `"Hanoi"` |
| `created_at` | timestamptz | Default `now()` |
| `updated_at` | timestamptz | Default `now()` |

---

### 4.6. presentations

A presentation entity. Stores the current (latest) version's metadata. Pages of the current version are in `presentation_pages`.

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid PK | Auto-generated UUID |
| `conversation_id` | uuid FK | Conversation this presentation belongs to |
| `topic` | text | Title/topic of the presentation |
| `total_pages` | integer | Number of pages in the current version |
| `version` | integer | Current version number; starts at 1, increments on each edit |
| `metadata` | jsonb | Stores `user_request` (the request that created this version) |
| `created_at` | timestamptz | Default `now()` |
| `updated_at` | timestamptz | Updated on every edit |

---

### 4.7. presentation_pages

Pages of the **current version** of a presentation. Replaced entirely on each edit.

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid PK | Auto-generated UUID |
| `presentation_id` | uuid FK | Presentation this page belongs to |
| `page_number` | integer | 1-based page number |
| `html_content` | text | Full HTML of the slide (1280×720) |
| `page_title` | text | Title of the page; nullable |
| `created_at` | timestamptz | Default `now()` |

---

### 4.8. presentation_versions

Archived previous versions of a presentation. Created automatically every time a presentation is edited.

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid PK | Auto-generated UUID |
| `presentation_id` | uuid FK | Presentation this version belongs to |
| `version` | integer | Version number (archived) |
| `total_pages` | integer | Number of pages in this version |
| `user_request` | text | The user's request that triggered this version |
| `created_at` | timestamptz | When this version was archived |

---

### 4.9. presentation_version_pages

Pages belonging to an archived version of a presentation.

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid PK | Auto-generated UUID |
| `version_id` | uuid FK | Archived version this page belongs to |
| `page_number` | integer | 1-based page number |
| `html_content` | text | Full HTML of the slide at time of archival |
| `page_title` | text | Title of the page; nullable |

---

### 4.10. token_blacklist

Revoked refresh tokens. When a user signs out, their refresh token's JTI is added here so it cannot be reused.

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid PK | Auto-generated UUID |
| `token_jti` | varchar(255) | JWT ID claim (`jti`) — unique per token |
| `user_id` | uuid FK | Owner of the revoked token |
| `token_type` | varchar(20) | Token type, e.g. `'refresh'` |
| `expires_at` | timestamptz | When the token naturally expires; used for cleanup |
| `blacklisted_at` | timestamptz | Default `now()` |

---

### 4.11. password_reset_tokens

One-time tokens sent via email for password reset. Token expires in 15 minutes and can only be used once.

| Column | Type | Description |
|--------|------|-------------|
| `id` | uuid PK | Auto-generated UUID |
| `token` | varchar(64) | Unique random token string |
| `user_id` | uuid FK | User requesting the reset |
| `expires_at` | timestamptz | Expiry time (15 minutes after creation) |
| `used_at` | timestamptz | When the token was used; NULL if not yet used |
| `created_at` | timestamptz | Default `now()` |

## 5. Database Relationships

### 5.1. One-to-Many (1:N)

| Parent | Child | Notes |
|--------|-------|-------|
| `users` | `conversations` | A user has many conversations |
| `users` | `user_facts` | A user has many saved facts |
| `users` | `token_blacklist` | A user can have many revoked tokens |
| `users` | `password_reset_tokens` | A user can request multiple reset tokens |
| `conversations` | `messages` | A conversation has many messages |
| `conversations` | `presentations` | A conversation can have many presentations |
| `presentations` | `presentation_pages` | A presentation has many current pages |
| `presentations` | `presentation_versions` | A presentation has many archived versions |
| `presentation_versions` | `presentation_version_pages` | An archived version has many pages |

### 5.2. One-to-One (1:1)

| Parent | Child | Notes |
|--------|-------|-------|
| `conversations` | `conversation_summaries` | At most one summary per conversation (UNIQUE constraint) |

### 5.3. Self-referencing / Special

| Table | Column | References | On Delete |
|-------|--------|------------|-----------|
| `conversations` | `active_presentation_id` | `presentations.id` | `SET NULL` — conversation keeps existing if active presentation is deleted |

### 5.4. Cascade Delete Strategy

All foreign keys (except `active_presentation_id`) use `ON DELETE CASCADE`. This means:

```
  DELETE users WHERE id = X
         │
         ▼  CASCADE
  DELETE conversations WHERE user_id = X
         │
         ├── CASCADE → messages
         ├── CASCADE → conversation_summaries
         └── CASCADE → presentations
                           │
                           ├── CASCADE → presentation_pages
                           └── CASCADE → presentation_versions
                                             │
                                             └── CASCADE → presentation_version_pages
```

Deleting a user removes all their data automatically at the database level — no manual cleanup required in application code.

## 6. Database Constraints

### 6.1. Primary Keys
All tables use `uuid` primary key with `DEFAULT gen_random_uuid()` (via `pgcrypto` extension).

### 6.2. Unique Constraints

| Table | Column(s) | Meaning |
|-------|-----------|---------|
| `users` | `email` | No duplicate email addresses |
| `conversation_summaries` | `conversation_id` | At most one summary per conversation |
| `user_facts` | `(user_id, key)` | A user cannot have two facts with the same key |

| `presentation_versions` | `(presentation_id, version)` | No duplicate version numbers within a presentation |

| `token_blacklist` | `token_jti` | Each token can only be blacklisted once |
| `password_reset_tokens` | `token` | Each reset token string is globally unique |

### 6.3. Check Constraints

| Table | Constraint | Rule |
|-------|------------|------|
| `messages` | `messages_role_check` | `role` must be `'user'`, `'assistant'`, or `'system'` |
| `messages` | `messages_intent_check` | `intent` must be `'PPTX'`, `'GENERAL'`, or `NULL` |
| `password_reset_tokens` | `chk_used_before_expiry` | `used_at IS NULL OR used_at <= expires_at` — token cannot be marked used after it has expired |

| Index | Table | Columns | Purpose |
|-------|-------|---------|---------|
| `idx_presentation_pages_number` | `presentation_pages` | `(presentation_id, page_number)` | Fetch pages of a presentation ordered by page number |

| `idx_version_pages_number` | `presentation_version_pages` | `(version_id, page_number)` | Fetch pages of an archived version ordered by page number |

### 7.3. Partial Index

A special index that only indexes rows matching a condition — smaller and faster than a full index.

```sql
CREATE INDEX idx_messages_working_memory
    ON public.messages(conversation_id, is_in_working_memory, created_at)
    WHERE is_in_working_memory = true;
```

**Why:** The most frequent message query is loading the working memory window for the LLM — only messages where `is_in_working_memory = true`. As conversations grow longer, most messages get summarized and excluded (`is_in_working_memory = false`). This partial index ensures the query only scans the active subset, not the full message history.
